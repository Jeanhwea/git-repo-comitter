# 项目结构

## 数据结构

### AppConfig

```typescript
interface AppConfig {
  endpoint: string;   // API 端点
  apiKey: string;     // API 密钥
  llm: LLMConfig;     // 大模型参数
}

interface LLMConfig {
  model: string;            // 模型名称
  temperature: number;      // 生成温度
  maxInputTokens: number;   // 最大输入 token 数
  maxOutputTokens: number;  // 最大输出 token 数
}
```

### CliError

```typescript
class CliError extends Error {
  exitCode: number;  // 进程退出码
}
```

## 项目架构

```
src/
├── index.ts              # 入口：启动 CLI
├── cli.ts                # 命令行解析与错误处理
├── errors.ts             # 自定义错误类型
├── commands/
│   ├── init.ts           # grc init — 交互式配置
│   └── commit.ts         # grc（默认）— 生成提交信息并提交
├── config/
│   ├── types.ts          # AppConfig / LLMConfig 类型与默认值
│   └── loader.ts         # 读取与保存 ~/.grc/config.json
├── git/
│   ├── runner.ts         # 统一 Git 命令执行器（execFileSync）
│   ├── diff.ts           # 获取 staged/unstaged diff
│   └── commit.ts         # git add / git commit
├── llm/
│   ├── client.ts         # 调用 OpenAI 兼容 API 生成提交信息
│   ├── checker.ts        # Conventional Commits 格式校验
│   └── response.ts       # LLM 响应内容提取
└── utils/
    ├── index.ts          # 工具函数入口
    └── cli.ts            # 交互式输入（readline）
```

### 数据流

```
用户运行 grc
  → cli.ts 解析参数
  → commands/commit.ts
    → config/loader.ts 读取 ~/.grc/config.json
    → git/diff.ts 获取 diff
    → llm/client.ts 调用 LLM API
      → llm/checker.ts 校验格式
      → llm/response.ts 提取响应内容
    → git/commit.ts 执行提交
```
