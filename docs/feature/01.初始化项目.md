# 项目初始化

## 项目概述

本项目是一个基于 TypeScript 的前端工具，用于调用大语言模型（采用 OpenAI 兼容格式），管理 Git 差异（diff），自动生成 Git 提交信息（commit message），并执行提交操作。

## 技术栈

- 编程语言：TypeScript
- 构建工具：Vite
- 包管理器：npm
- 代码格式化：Prettier + ESLint
- Git 钩子：husky + lint-staged

## 功能需求

### 核心功能

1. **Git 差异管理**
   - 读取当前 Git 仓库的变更内容（diff）
   - 支持暂存区和工作区的差异提取

2. **大模型调用**
   - 调用外部大语言模型 API 生成提交信息（OpenAI 兼容格式）
   - 模型配置通过 `~/.grc/config.json` 管理

3. **配置系统**
   - 配置文件：`~/.grc/config.json`
   - 可配置项包括：
     - `endpoint`：API 服务地址
     - `apiKey`：API 密钥
     - `llm.model`：模型名称
     - `llm.temperature`：生成温度
     - `llm.maxInputTokens`：最大输入 token 数
     - `llm.maxOutputTokens`：最大输出 token 数

4. **Git 提交**
   - 自动执行 `git commit` 操作
   - 使用生成的提交信息

## 初始化步骤

1. **创建项目目录并初始化 npm**

   ```bash
   mkdir git-repo-committer
   cd git-repo-committer
   npm init -y
   ```

2. **初始化 TypeScript 配置**

   ```bash
   npm install --save-dev typescript
   npx tsc --init
   ```

3. **安装 Vite 及相关依赖**

   ```bash
   npm install --save-dev vite @types/node
   ```

4. **安装代码格式化工具**

   ```bash
   npm install --save-dev prettier eslint eslint-config-prettier @typescript-eslint/parser @typescript-eslint/eslint-plugin
   ```

5. **安装 Git 钩子工具**

   ```bash
   npm install --save-dev husky lint-staged
   npx husky init
   ```

6. **安装运行时依赖**

   ```bash
   npm install commander openai
   ```

7. **创建配置文件**

   - 创建 `.eslintrc.cjs`、`.prettierrc` 配置规则
   - 创建 `vite.config.ts` 定义构建目标

8. **实现核心模块**

   - `src/config/loader.ts`：读取和保存 `~/.grc/config.json`
   - `src/git/runner.ts`：统一 Git 命令执行
   - `src/git/diff.ts`：执行 `git diff` 并解析结果
   - `src/llm/client.ts`：使用 OpenAI SDK 调用兼容 API
   - `src/commands/commit.ts`：组装并执行提交流程

9. **完善 package.json 元数据**

## 发布计划

项目完成后将发布到 npm registry，作为命令行工具（CLI）供开发者使用。发布步骤：

1. 确保 `package.json` 中已填入正确的 GitHub 仓库地址和元信息
2. 运行 `npm run build` 构建产物
3. 执行 `npm publish` 发布

## 注意事项

- 确保 `.gitignore` 包含 `node_modules`、`dist` 等目录
- 配置文件 `~/.grc/config.json` 中包含 API Key，注意不要泄露
- 使用 `openai` 包时需配置 `baseURL` 为 `endpoint` 的值
- 代码提交前会通过 husky + lint-staged 自动执行 ESLint 和 Prettier 格式化
